---

## Inline References

Put references after the change text on the same line, wrapped in parentheses.

```md
- Fix broken release badge link (#18)
```

---

## Choosing The Best Reference

Prefer the most useful starting point:

1. Pull request when it contains issue context, implementation, review, and discussion.
2. Issue when it explains the user problem better than the implementation.
3. Commit when no better reference exists.
4. External ticket or spec when it is the clearest source of context.

---

## Multiple References

Group references of the same type.

```md
- Fix template validation for empty issue bodies (#24, #27)
```

When too many references are available, include only the best starting point.

---

## Authors

Authors may appear after references.

```md
- Add release checklist (#12) (Ada Lovelace)
```

For bot-authored changes, prefer the human reviewer or merger.
